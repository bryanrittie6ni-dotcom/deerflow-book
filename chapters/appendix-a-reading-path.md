# 附录 A　阅读路径指南

本书涵盖 DeerFlow 的架构设计、核心实现与生产运维，内容较多。根据你的角色和目标，推荐以下三条阅读路径。

## 路径一：入门开发者

**推荐章节：第 1-3 章，然后第 4-5 章**

如果你是第一次接触 DeerFlow，建议从第 1 章的项目概览开始，了解 DeerFlow 是什么、解决什么问题。第 2 章和第 3 章带你完成环境搭建和第一次运行，建立直观体验。有了基础认知后，阅读第 4-5 章深入理解工作流的核心概念和对话流转机制。这条路径的目标是让你能够独立配置和使用 DeerFlow，并对系统有一个完整的全局认知。

## 路径二：架构师 / 深度开发者

**推荐章节：第 6-11 章**

如果你已经熟悉 DeerFlow 的基本用法，想要深入理解或定制扩展系统，重点阅读第 6 章 Lead Agent 的调度逻辑、第 7-8 章的工具与 MCP 集成、第 9-11 章的 Sub-Agent 架构和 Skill 系统。这些章节覆盖了 DeerFlow 多 Agent 协作的核心设计——从任务分解、工具调用到代码执行的完整链路。读完这些章节，你将具备为 DeerFlow 开发自定义 Agent、编写新 Skill 或集成第三方 MCP 服务器的能力。

## 路径三：运维 / DevOps 工程师

**推荐章节：第 14-15 章，第 20-22 章**

如果你的职责是部署和运维 DeerFlow，可以直接跳到第 14-15 章了解沙箱隔离与安全机制，然后阅读第 20-22 章的配置体系、模型适配和部署指南。第 20 章的配置速查和附录 B 的字段速查表是日常运维的高频参考资料。第 22 章详细讲解了 Docker Compose 部署、Nginx 配置、LangSmith 追踪和性能调优，覆盖了从搭建到监控的全流程。
